# TB Order Sync 公司同事使用说明

适用范围：
- Windows 电脑
- macOS 电脑

当前版本：
- `0.4.2`

这套工具目前负责两件事：
- 自动计算毛利
- 自动匹配退款单号，并在毛利表中写入 `已退款` + 整行红字

---

## 1. 你会收到什么

建议交付给同事的是“打包好的运行目录”，不要让同事自己搭 Python 环境。

分发包里至少应包含这些文件：
- Windows:
  - `sync_service.exe`
  - `启动.bat`
  - `.env.example`
  - `快速开始.txt`
- macOS:
  - `sync_service`
  - `启动.command`
  - `.env.example`
  - `快速开始.txt`

如果没有打包好的可执行文件，也可以发源码目录，但更适合技术同事。

---

## 2. 第一次使用

### Windows

1. 解压整个项目目录到本地，例如桌面。
2. 双击 `启动.bat`
3. 首次没有配置时，程序会自动进入配置向导
4. 按提示填写：
   - 腾讯文档 `Client ID`
   - 腾讯文档 `Open ID`
   - 腾讯文档 `Access Token`
   - 毛利表链接或 File ID
   - 客户退款表链接或 File ID
5. 配置完成后，先执行一次 `tb check`
6. 自检通过后，再执行 `tb all`

### macOS

1. 解压整个项目目录到本地，例如桌面。
2. 双击 `启动.command`
3. 如果系统提示“无法打开”，请右键文件，选择“打开”
4. 首次没有配置时，程序会自动进入配置向导
5. 按提示填写：
   - 腾讯文档 `Client ID`
   - 腾讯文档 `Open ID`
   - 腾讯文档 `Access Token`
   - 毛利表链接或 File ID
   - 客户退款表链接或 File ID
6. 配置完成后，先执行一次 `tb check`
7. 自检通过后，再执行 `tb all`

## 3. 最推荐的使用方式

### 方式 A：双击启动

- Windows：双击 `启动.bat`
- macOS：双击 `启动.command`

进入控制台后可直接选择：
- 执行全部任务
- 模拟执行
- 启动守护
- 配置向导
- 配置检查

### 方式 B：命令行启动

如果电脑已经装好 Node/npm，并且已经全局安装：

```bash
npm install -g tb-order-sync

tb
tb setup
tb check
tb all
tb daemon start
tb daemon status
```

macOS 通过 npm 全局安装时，运行环境会自动部署到：

```text
~/Library/Application Support/tb-order-sync/
```

其中包括：
- `.env`
- `state/`
- `.venv/`

## 4. 日常使用步骤

### 手动执行一次

```bash
tb all
```

功能包括：
- 重新计算毛利表中的毛利
- 从客户退款表读取单号
- 匹配成功后，在毛利表写入 `已退款`
- 对应整行文字标红

### 先演练，不写入

```bash
tb all --dry-run
```

### 仅执行毛利计算

```bash
tb gp
```

### 仅执行退款匹配

```bash
tb rm
```

---

## 5. 后台持续运行

如果希望工具长期在后台自动执行：

### 启动后台守护

```bash
tb daemon start
```

### 查看守护状态

```bash
tb daemon status
```

### 查看后台日志

```bash
tb daemon logs --lines 80
```

### 停止后台守护

```bash
tb daemon stop
```

---

## 6. 开机后自动运行

如果希望员工每次开机登录后自动启动后台任务：

### 启用登录自启

```bash
tb daemon autostart-enable
```

### 查看登录自启状态

```bash
tb daemon autostart-status
```

### 关闭登录自启

```bash
tb daemon autostart-disable
```

说明：
- Windows 使用“任务计划程序”
- macOS 使用“LaunchAgent”

---

## 7. 配置时需要准备什么

必须准备：
- 腾讯文档 `Client ID`
- 腾讯文档 `Open ID`
- 腾讯文档 `Access Token`
- 毛利表链接
- 客户退款表链接

当前运行链路里：
- `Client Secret` 不是必须项
- 如果后续接自动刷新 Token，再补也可以

---

## 8. 如何确认系统是正常工作的

执行：

```bash
tb check
```

这一步会检查：
- 必填配置是否存在
- 状态目录是否可写
- 毛利表是否可读
- 客户退款表是否可读

只有 `tb check` 通过后，再建议正式运行。

---

## 9. 运行成功后你应该看到什么

### 毛利计算成功后

- 毛利表 `G` 列会写入毛利结果
- 如果数据异常，会写入 `数据异常`
- 如果一行 `C/D/E` 都为空，则 `G` 会保持为空，不会误写 `数据异常`

### 退款匹配成功后

- 如果退款表单号命中毛利表单号：
  - 毛利表 `I` 列写入 `已退款`
  - 对应整行文字变红

注意：
- 当前高亮效果是“整行红色文字”
- 不是“背景填充红色”

## 10. 常见问题

### 1）双击没反应

- Windows：确认是否被安全软件拦截
- macOS：右键文件，选择“打开”

### 2）提示缺少配置

执行：

```bash
tb setup
```

### 3）提示连接失败

先执行：

```bash
tb check
```

重点检查：
- `Access Token` 是否过期
- `Open ID` 是否正确
- 表格链接是否正确
- 当前账号是否有表格访问权限

### 4）提示限流或稍后重试

这是腾讯文档接口限流，建议：
- 不要连续快速重复执行
- 等一会儿再重试

### 5）后台看起来没动静

执行：

```bash
tb daemon status
tb daemon logs --lines 80
```

---

## 11. 日志和状态文件在哪里

都在 `state/` 目录下：

- `state/scheduler.console.log`
  - 后台运行日志
- `state/sync_state.json`
  - 增量状态
- `state/last_run.json`
  - 最近一次执行结果摘要

---

## 12. 推荐交付口径

给公司同事时，建议你这样说：

1. 双击启动文件
2. 第一次先做配置
3. 再执行一次 `配置检查`
4. 没问题后执行“全部任务”
5. 如需长期自动运行，就启动守护并开启登录自启

最简流程就是：

```text
双击启动 → 配置向导 → 配置检查 → 执行全部任务 → 开启后台守护
```
